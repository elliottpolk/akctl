# 2026-04

## 2026-04-06

- **Issue**: New skills (like `gemini-bridge`) and workflows were not being pulled during `akctl sync`.
- **Root Cause**: The `Manifest` struct in `internal/kernel/kernel.go` only tracked `Agents`. `ComponentPathsFromManifest` (formerly `AgentPathsFromManifest`) only extracted paths for agents. This resulted in `akctl sync` not being aware of the full set of component directories to download or protect.
- **Decision**: Expanded `Manifest` and `Component` structs to include `Workflows` and `Skills`. Renamed path extraction logic to be component-agnostic.
- **Action**: 
    - Updated `internal/kernel/kernel.go`: `Manifest` now has `Workflows` and `Skills`.
    - Renamed `AgentPathsFromManifest` to `ComponentPathsFromManifest`.
    - Updated `internal/sync/sync.go` and `internal/sync/sync_test.go` to use the new component-agnostic logic.
    - Verified with `TestMergeManifest_upstreamRaw` and expanded `TestUserOwned` to include skill paths.
- **Result**: `akctl sync` now correctly pulls all components (agents, workflows, skills) from the upstream kernel.
