# 2026-04

## 2026-04-06

- **Issue**: New skills (like `gemini-bridge`) and workflows were not being pulled during `akctl sync`.
- **Root Cause**: The `Manifest` struct in `internal/kernel/kernel.go` only tracked `Agents`. `ComponentPathsFromManifest` (formerly `AgentPathsFromManifest`) only extracted paths for agents. This resulted in `akctl sync` not being aware of the full set of component directories to download or protect.
- **Decision**: Expanded `Manifest` and `Component` structs to include `Workflows` and `Skills`. Renamed path extraction logic to be component-agnostic.
- **Action**: 
    - Updated `internal/kernel/kernel.go`: `Manifest` now has `Workflows` and `Skills`.
    - Renamed `AgentPathsFromManifest` to `ComponentPathsFromManifest`.
    - Updated `internal/sync/sync.go` and `internal/sync/sync_test.go` to use the new component-agnostic logic.
    - Verified with `TestMergeManifest_upstreamRaw` and expanded `TestUserOwned` to include skill paths.
- **Result**: `akctl sync` now correctly pulls all components (agents, workflows, skills) from the upstream kernel.

- **Issue**: `init` and `sync` commands lacked visual feedback for long-running fetch operations and used inconsistent styling for warnings and paths.
- **Decision**: Introduce a `ProgressReporter` interface and a `bubbletea`-based TUI implementation. Unify CLI styling using a central `internal/ui` package.
- **Action**:
    - Created `internal/ui` package with `ProgressReporter` (using `bubbletea` and `bubbles/progress`) and common `lipgloss` styles.
    - Refactored `internal/kernel/kernel.go` to support progress reporting in `Fetch`.
    - Updated `cmd/akctl/main.go` and `internal/sync/sync.go` to use `TeaProgressReporter` during kernel fetching.
    - Unified warning and path display styles across `internal/setup` and `internal/sync` using the new `ui` package.
    - Updated `go.mod` and `go.sum` to include `bubbles` and its dependency `harmonica`.
- **Result**: `akctl` now provides a polished TUI with progress bars during `init` and `sync`, and maintains a consistent visual identity.

- **Issue**: `akctl sync` was overwriting all core files regardless of whether they had changed. TUI progress bars were not being cleared from the terminal, and confirmation dialogs lacked visual separation.
- **Decision**: Implement content-based change detection for sync and refine TUI behavior for a cleaner user experience.
- **Action**:
    - Updated `internal/sync/sync.go` to compare local and upstream file contents, including a dry-run merge for `manifest.yml`.
    - Modified `internal/sync/sync.go:Run` to report if everything is already up to date.
    - Updated `internal/ui/ui.go` to use `tea.WithOutput(os.Stderr)` and `prog.Wait()` to ensure progress bars are properly cleared.
    - Updated all `huh.Form` instances in `internal/sync` and `internal/setup` to use `huh.ThemeCharm()` for a "popup-like" appearance.
- **Result**: `akctl sync` is now significantly faster and more accurate, only touching changed files. The TUI is cleaner and more visually distinct.
